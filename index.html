Backbone.js

Viewオブジェクトの利用

  var 変数 = Backbone.View.extend(...設定情報...);
  Viewは、Backbone.Viewオブジェクトの「extend」メソッドを使って作成
  このオブジェクトを元にnewして操作するオブジェクトを作成します
  var 変数 = new オブジェクト(...設定情報...);

renderメソッドについて

  {
    render:finction(){....}
  }
  「render」は画面表示のレンダリングを行うためのメソッド

$elとel

  「$el」は、Viewオブジェクトよって生成される表示エレメントを設定したjQueryオブジェクト
  「el」は、表示するエレメントのオブジェクトそのものを示すプロパティ（jQueryを使わない場合、こっちを利用）
  ※Backbone.jsではjQurtyの利用を前提に作られている

Viewの活用

構造化されたコンンテンツの操作

構造化されたHTMLのコンテンツをViewから更新する

intializeによる初期化

「intialize」メソッドを使い。Viewの初期化処理をする
操作する対象を整理して、使いやすくする。
よく使うエレメントをViewプロパティに登録して扱えるようにする

イベントの利用（eventプロパティ）

  Viewでは表示する対象となるエレメントのイベントを設定することができる
  event: {
    イベント名: function(event){...処理...};
  }

新たなエレメントの生成

  タグの名前やclass、id、その他の属性の値などをプロパティとして用意しておく
  「el」プロパティは用意しないように注意する

テンプレートの利用

  <script>タグの中にHTMLのタグを記述して作成する
  <script id="IDの指定">
    ...表示する内容...
  </script>

  以下のようにしてView内でロードする
  var 変数 = _.template(...テンプレートの内容...)
  <%= 変数 %>でテンプレートに渡された値を出力させることができる

複数Viewの連結

elを指定することで特定要素に自動的に割り当てることができる
制御したい要素が複数ある場合は、その数だけViewを作ることができる
